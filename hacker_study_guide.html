Execution Path // Recommended Sequence
01
SETUP YOUR LAB
Install VirtualBox → Deploy Kali Linux → Configure host-only network → Snapshot your baseline. This is your sandbox. Never skip this.
WEEK 1
02
LINUX GRIND — OverTheWire Bandit
Complete Bandit levels 0–34. Do not look up answers until you've spent 20+ mins stuck. Keep a personal notes file. Learn pipes, grep, find, ssh, file permissions cold.
WEEKS 1–2
03
THEORY LAYER — HackerSploit
Watch the Ethical Hacking & Pentesting playlist. Pause and replicate every tool demo in your Kali VM. Focus: recon, Nmap, Nikto, Metasploit basics, enumeration methodology.
WEEKS 2–4
04
WEB ATTACK CORE — PortSwigger Academy
Work through Server-Side topics first: SQLi → Authentication → Path Traversal → SSRF → Command Injection. Solve every lab before reading the solution. Burp Intercept every request.
WEEKS 3–8
05
REAL MACHINES — HTB Starting Point + VulnHub
Begin with HTB Starting Point (guided). Then move to VulnHub: Kioptrix 1 → 2 → 3, then DC-1, Mr-Robot. For each machine: enumerate first, attack second, document everything.
WEEKS 5–12+
06
EXPANSION — AD LABS + ADVANCED TOPICS
Build a free Active Directory home lab (freeCodeCamp AD video). Study lateral movement, pass-the-hash, Kerberoasting. Begin tracking gaps. Consider TryHackMe or HTB subscription at this stage.
MONTH 3+

Operator Notes // Key Commands
root@kali:~# nmap -sC -sV -oA initial 10.10.10.X # Always save output
→ Start every machine with a full port scan. Document everything. Enumerate before you exploit.

root@kali:~# gobuster dir -u http://TARGET -w /usr/share/wordlists/dirb/big.txt
→ Bruteforce directories on every web service you find. Hidden paths = hidden attack surface.

root@kali:~# searchsploit [service name] [version]
→ Cross-reference every identified service version against Exploit-DB before manual testing.

root@kali:~# python3 -m http.server 8000 # Quick file transfer server
→ Serve exploit scripts and post-exploitation tools to target machines over HTTP.

⚠ Rule: If you're stuck for 2h, read a hint. If 4h, read a walkthrough. Don't spin forever.
⚠ Always document: screenshots, commands run, findings, flags captured. Build the habit now.
