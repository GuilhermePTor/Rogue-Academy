MODULE_03 // WEEKS 4–5 // DATATALKSC.LUB MLOPS ZOOMCAMP

Orchestration & ML Pipelines

Prefect, Mage, and turning notebooks into production pipelines — ~8–10 hrs
A training script you run manually is not a pipeline — it's a ritual. Pipelines encode your workflow as code: each step is a function with defined inputs and outputs, retries on failure, logging, scheduling, and visibility into what ran when and why. The Zoomcamp covers Prefect as the primary tool and Mage as an alternative.

Model Deployment

Online serving, batch scoring, and streaming — ~10–12 hrs
Deployment has three modes, each with different latency, infrastructure, and cost trade-offs. Online serving answers requests in milliseconds. Batch scoring processes millions of records overnight. Streaming sits in between — predictions triggered by events flowing through Kafka or Kinesis. The Zoomcamp covers all three.

MODULE_05 // WEEK 8 // DATATALKSC.LUB MLOPS ZOOMCAMP

Model Monitoring

Data drift, concept drift, Evidently, Grafana, Prometheus — ~8–10 hrs
Deploying a model is not the end — it's the beginning of a different problem. Models degrade silently. Your input data distribution drifts. User behaviour changes. Without monitoring, you find out the model is broken when someone complains. With monitoring, you find out three days before anyone notices. The Zoomcamp covers Evidently for drift detection and Grafana + Prometheus for dashboards.
